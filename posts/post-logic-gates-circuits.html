Get to know about logic gates and build a calculator

I got interested in logic gates during the summer, so I decided to try to build a calculator with them. This blog records some of the problems I encountered and will hopefully help someone who is also interested in logic gates. If you want to have a copy of the calculator, you can go to here.

Half Adder

OK, let's get right into it. The picture below is the structure of a half adder.



half adders add two binary digits together. the top bulb shows the output at this digit and the bottom bulb shows the carrier. However, when we are doing addition, we need to add three inputs together in a single digit: the carrier, digit from numA and digit from numB.


Full Adder

Full adder is consisted of two half adders. It can take in three inputs can add them together.



The bottom bulb is carrier.


Binary Addition

With half adders and full adders, we are able to perform binary additions. The image shown below is an example of a 8-bit adder. remember that when you add two 8-bit number, you can have a 9 bit result.



half subtractor

The structure of half subtractor is very similar to that of half adder. you simply put a NOT gate after input A.



Note when the botom bulb is lit, it indicates that we need to borrow from the next digit.


Full subtractor

Full subtractor is consisted of two half subtractors. Since when we are doing binary subtraction, we need to subtractor two digits from the digit from numA, the borrow and the digit of numB.


Again, the top bulb show the current digit of the result and the bottom shows the borrow from the next digit.


Binary Subtraction


We can make binary subtraction circuits with half and full subtractors. However, diferent from binary addition, we have one more step to go. When the result is negative, the top full subtractor think that it can still borrow from higher digits, which does not exist. We need to correct this by doing some inversion. When you invert a binary number, you turn all the digits that are 1 to 0 and digits that are 0 to 1. for example, 1001 to 0110. Then we add one to it, it will become the positive number we want. for example, when the result should be -1, the calculator will show 1111. we then turn it into 0000 and add 1 to it to get 1.

Remember that all the above only take place when the result is negative, which take place when the borrow output of the topmost subtractor is one. Thus, we can use XOR gate to do the inversion.



multiplication

the implementation of multiplication is pretty staight forward. try to do a 3 bit by 3 bit multuplication on paper, and that's exactly how you make the circuit. you use and gate to multiply two digit together because if either of the digit is zero, the result is zero. then, label each AND gate with its level, meaning which digit should we add it into. then, for each result digit, we add all the carries and digits together. the image below is how a 3 bit by three bit multiplier is structured.



using the same idea, we can build multipliers that can handle larger inputs.
