we can solve this problem using recursion with memoization, or dynamic programming. Any given string can be formed in four ways. we recursively search for each of them, and sum them toather to get the ans for the current string. since for equal strings, they surely have the same answer, so we can store the value we got for a specific string in an hashmap, thus preventing us from calculating the same thing again and again.

Another thing is that when we finished the resursive function, the return value actually counted the original string as one of the ways to obtain the given string. thus to get the actual answer, we need to subtract one from that.
